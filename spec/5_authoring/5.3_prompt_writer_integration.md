# 5.3 Prompt Writer Integration

## Goal

Integrate prompt writing with the new synthesis-first pipeline.

Prompt writing is no longer driven primarily by:
- clip plans
- raw analysis

Instead it should be driven by:

- character bibles (Phase 7)
- environment bibles (Phase 8)
- scene contracts (Phase 9)
- shot packages (Phase 10)

---

## New Architecture Role

### Old

analysis → clip plan → prompt writing

### New

analysis
→ bibles
→ scene contracts
→ shot packages
→ prompt writing

---

## Prompt Inputs (Updated)

Prompt writer should consume:

- character bible JSON
- environment bible JSON
- scene contract JSON
- shot package JSON

It should no longer rely on:

- raw character breakdowns
- raw environment breakdowns
- raw scene decomposition

except as fallback or traceability.

---

## Prompt Responsibilities

### Character reference prompts

Now sourced from:
- character bible (not raw extraction)

### Environment reference prompts

Now sourced from:
- environment bible

### Shot prompts (replacing clip prompts)

Now sourced from:
- shot package
- scene contract

---

## Compatibility Layer

During migration:

- prompt writer may accept legacy clip plans
- but should internally normalize them into shot-like structures

---

## Required Changes

### Add new input loader layer

```text
orchestrator/prompt_inputs.py
```

Responsibilities:
- load bibles
- load scene contracts
- load shot packages
- normalize into prompt-ready structure

---

### Update prompt writer

```text
orchestrator/prompt_writer.py
```

Responsibilities:
- generate prompts from canonical structures
- preserve existing output format

---

## Prompt Schema (unchanged externally)

Keep:
- existing prompt file structure
- runner compatibility

Change:
- where data comes from

---

## Acceptance

- prompt writer works without raw analysis dependency
- prompts are consistent across scenes
- characters remain visually stable
