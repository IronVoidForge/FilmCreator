# 5.3 Prompt Writer Integration

## Goal

Plug the authoring layer into the manual runner without changing the runner contract.

## Current Implementation Scope

- `lmstudio-check` is now implemented as a local connectivity and model-resolution smoke test.
- `write-prompts` is now implemented for clip-local authoring prompt families:
  - `scene_stage`
  - `keyframe`
  - `still_fix`
  - `cut_motion`
- The current implementation rewrites the canonical prompt-package files in place and records those paths back into `clip_state.json`.
- Shared character-reference and environment-reference prompt writing are still planned work.
- Scene analysis and scene-wide clip planning remain separate planned authoring steps.

## Local API Contract

- Prompt writing uses the same local LM Studio API family the novel orchestrator uses: an OpenAI-compatible local endpoint.
- Proposed runtime settings:
  - `FILMCREATOR_LMSTUDIO_BASE_URL`
  - `FILMCREATOR_LMSTUDIO_MODEL`
  - `FILMCREATOR_LMSTUDIO_TIMEOUT_SECONDS`
- Prompt writing should fail clearly when LM Studio is unreachable, the model is missing, or the response cannot be converted into the required file schema.

## Rules

- Authoring writes Markdown prompt packages into `03_prompt_packages/`.
- Prompt writer output must match the schema from `1.4`.
- Prompt packages reference analysis files in their `Sources` section.
- Prompt writer output should cover both shot openers and continuation intervals derived from the clip plan.
- For continuation stages, prompt packages must state the expected source-frame behavior in `Inputs` and `Continuity Notes`, even when the runner resolves the exact file path from state.
- Authoring may use a local LLM runtime via a compatible API, but the runner only depends on local files and local render services.
- The runner does not care whether the prompt was written by hand or by an LLM.
- Scene-stage prompt writing remains an authoring step, not a render stage.
- Scene-stage prompt packages should not pull reviewed video candidates or other runtime media outputs into `required_refs`.
- Scene-stage prompt packages should describe staging intent and planning context, not behave like a render-ready workflow input map.
- `required_refs` and `optional_refs` should describe workflow slot expectations such as `image_1`, `image_2`, or `source_frame`, not concrete reviewed media file paths.
- `workflow_type` belongs in the dedicated heading, not duplicated again inside `Inputs`.
- The LLM planning pass must decide, per clip, which shot-start category the opening keyframe belongs to.
- Generated keyframe, still-fix, and cut-motion prompt bodies should avoid proper nouns and prefer descriptive noun phrases.
- `duration_seconds` belongs in `Inputs`, not in the final prompt body text.
- Cut-motion prompts should describe visible motion from the approved opening frame instead of re-describing every static detail already present in that frame.
- Cut-motion prompts should preserve the approved keyframe's lighting, color temperature, and general grade by default unless the clip plan explicitly calls for a visible look transition.
- Cut-motion prompts should not introduce an unexplained cool or blue cast when the approved keyframe establishes a different look.
- Normal cut-motion prompts should target one short image-to-video generation, not a stitched long-form continuation workflow by default.
- If a clip plan declares multiple motion segments, authoring should make that explicit instead of hiding the extra duration inside one oversized motion prompt.
- LongLook-specific motion prompting is reserved for clips explicitly marked as extended cuts.
- The LLM should prefer opening-keyframe strategies that do not require prior video completion unless the shot genuinely needs direct carry-forward continuity.
- The LLM should treat `continuous_follow` as rare and should prefer:
  - `reframe_same_moment`
  - `reblock_same_scene`
  - `insert`
  - `cutaway`
- The LLM should be able to mark a review fallback path where a failed next-shot keyframe is retried by using the previous approved video last frame as an image-to-image camera-repositioning source.
- The LLM should also be able to mark whether an optional post-keyframe identity-consistency assist should run for visible hero characters in the shot.
- Identity-consistency assist should prefer low-denoise reference-guided image-to-image using approved character-sheet refs rather than a destructive redraw.
- Multi-character identity correction should prefer region-aware reference conditioning when supported by the selected workflow family.
- If region-aware targeting is unavailable, the LLM should prefer conservative single-subject correction or no automatic assist rather than forcing a bad whole-frame rewrite.
- Hand or anatomy repair should be treated as a separate corrective intent from identity consistency.
- Any future hand-fix LoRA in video should remain experimental and opt-in until validated on the local Wan workflow family.

## Prompt Families To Write

- Shared character prompts
  - destination: `03_prompt_packages/characters/<asset_id>/<asset_id>_ref_prompt.md`
  - purpose: generate project-level character reference stills
- Shared environment prompts
  - destination: `03_prompt_packages/environments/<asset_id>/<asset_id>_ref_prompt.md`
  - purpose: generate project-level environment reference stills
- Scene-stage prompts
  - destination: `03_prompt_packages/scenes/<scene_id>/<clip_id>/<scene_id>_<clip_id>_scene_stage_prompt.md`
  - purpose: authoring-only staging intent for the cut
- Keyframe prompts
  - destination: `03_prompt_packages/keyframes/<scene_id>/<clip_id>/<scene_id>_<clip_id>_keyframe_prompt.md`
  - purpose: the exact visible state at cut start, based on the chosen starting-keyframe strategy
- Still-fix prompts
  - destination: `03_prompt_packages/fixes/<scene_id>/<clip_id>/<scene_id>_<clip_id>_fix_01_prompt.md`
  - purpose: corrective still-generation instructions after review
- Cut-motion prompts
  - destination: `03_prompt_packages/cut_motion/<scene_id>/<clip_id>/<scene_id>_<clip_id>_cut_motion_prompt.md`
  - purpose: visible subject, camera, and environment motion for the next short motion segment starting from the approved keyframe

## Prompt Responsibilities

- Character reference prompt
  - emphasize stable visual identity and costume-readable details
- Environment reference prompt
  - emphasize architecture, lighting, mood, and stable environmental facts
- Scene-stage prompt
  - explain staging intent, subject placement, and environmental context for the cut
- Keyframe prompt
  - describe the first approved still of the cut as a single frozen visible state
  - reflect the selected continuity mode and composition type for that clip
- Still-fix prompt
  - correct a reviewed problem without redesigning the entire shot
  - may target identity consistency, anatomy repair, or compositional cleanup
- Cut-motion prompt
  - describe motion, camera movement, and environmental change that begins at the approved keyframe
  - assume the keyframe already carries identity, composition, and styling unless the clip plan explicitly requires a new segment handoff

## Planned Commands

- `python -m orchestrator lmstudio-check`
  - verifies LM Studio connectivity and configured model
- `python -m orchestrator write-prompts <project_slug> --scene <scene_id>`
  - writes all prompt packages for every clip in one scene
- `python -m orchestrator write-prompts <project_slug> --scene <scene_id> --clip <clip_id>`
  - rewrites prompt packages for one clip
- `python -m orchestrator author-scene <project_slug> --scene <scene_id>`
  - later convenience command that chains analysis, clip planning, and prompt writing in one authoring pass

## Required Authoring Behavior

- Generated files should be reproducible enough that rerunning authoring refreshes the same canonical file paths rather than inventing new filenames.
- Prompt writing should preserve manual edits only when an explicit future merge policy exists; until then, the spec should assume full-file replacement on regeneration.
- Prompt writing should populate `Inputs` with:
  - `project_id`
  - `scene_id`
  - `clip_id`
  - `beat_id`
  - `duration_seconds`
  - `required_refs`
  - `optional_refs`
  - `visible_character_assets`
  - `look_continuity_policy`
  - `intended_lighting_change`
  - `composition_type`
  - `continuity_mode`
  - `starting_keyframe_strategy`
  - `dependency_policy`
  - `auto_advance_policy`
  - `fallback_strategy`
  - `consistency_assist_policy`
  - `consistency_assist_method`
  - `anatomy_repair_policy`
  - `consistency_targets`
  - `style_profile`
  - `batch_role`
  - `fix_of`
- Prompt writing should include `Sources` that point back to:
  - scene breakdowns
  - character breakdowns
  - environment breakdowns
  - clip roster
  - clip plan

## Handoff

- input:
  - analysis files under `02_story_analysis/`
  - LM Studio local API
  - existing project, scene, and clip IDs
- output:
  - Markdown prompt packages written into canonical `03_prompt_packages/` folders
  - no workflow patching yet
  - no ComfyUI render submission yet

## Acceptance

- Replacing manual prompt writing with local automated authoring requires no orchestrator redesign.
- A prompt-writing run can generate character, environment, scene-stage, keyframe, still-fix, and cut-motion prompt packages into their canonical folders.
- The runner can immediately consume generated prompt packages without schema translation or manual cleanup.
- The written cut-motion prompts clearly support the default short-cut image-to-video path while still leaving room for later explicit extended-cut workflows.
- The planning and prompt-writing layer can decide whether a clip should start from independent reframing, reblocking, insert/cutaway logic, or a rarer previous-video-based camera-reposition fallback.
- The planning and prompt-writing layer can also decide whether an optional identity-consistency or anatomy-repair assist should be attempted and can point that assist at the correct character refs.
- The planning and prompt-writing layer can also specify whether the motion stage should preserve the approved keyframe look or perform a planned lighting transition.
- In practice, character-to-scene mapping should be treated as a prerequisite before the still-fix stage becomes a primary workflow focus, because corrective prompts need to know which approved character refs belong in the current shot.
