# 5.1 Story Analysis Outputs

## Goal

Define the analysis files that authoring should produce before prompt writing.

## Local Runtime Contract

- Authoring talks to a local LM Studio instance through its OpenAI-compatible API.
- Rendering does not depend on LM Studio remaining open after authoring completes.
- Analysis output is file-first. The runner later consumes files, not live LLM responses.

## Required Outputs

- Project story summary
- Scene breakdowns grouped by scene ID
- Shot lists or clip rosters grouped by scene ID
- Character breakdowns keyed by asset ID
- Environment breakdowns keyed by asset ID
- Clip plans grouped by scene ID and clip ID, each with a 3-5 second interval breakdown

## Planned Files

- `02_story_analysis/story_summary/project_summary.md`
- `02_story_analysis/scene_breakdowns/<scene_id>.md`
- `02_story_analysis/character_breakdowns/<asset_id>.md`
- `02_story_analysis/environment_breakdowns/<asset_id>.md`
- `02_story_analysis/clip_plans/<scene_id>/<scene_id>_clip_roster.md`
- `02_story_analysis/clip_plans/<scene_id>/<clip_id>.md`

## Required Analysis Content

- `project_summary.md`
  - overall premise
  - major recurring characters
  - major recurring environments
  - continuity-sensitive visual rules
- `scene_breakdowns/<scene_id>.md`
  - scene purpose
  - emotional and dramatic function
  - ordered beats
  - likely shared refs needed for the scene
- `character_breakdowns/<asset_id>.md`
  - physical description
  - costume or silhouette rules
  - continuity-critical traits
  - useful descriptive noun phrases for later prompt writing
- `environment_breakdowns/<asset_id>.md`
  - architectural or geographic description
  - lighting and atmosphere rules
  - stable environmental elements
  - useful descriptive noun phrases for later prompt writing
- `clip_roster`
  - ordered clip IDs for the scene
  - one-line purpose for each clip
  - target duration for each clip
- `clip_plans/<scene_id>/<clip_id>.md`
  - shot or cut purpose
  - opening state at cut start
  - textual 3-5 second interval beats
  - continuity start mode
  - required and optional refs
  - expected prompt-package targets

## Rules

- Analysis files live under `02_story_analysis/`.
- Analysis first decomposes story scenes into shot-level clips.
- Each clip plan records the shot intent and the interval beats used by prompt writing.
- Analysis is descriptive input to prompt writing, not a replacement for prompt packages.
- Clip plans bridge scene intent into clip-local render units.
- Analysis may use proper nouns and asset IDs internally so humans can follow the files easily.
- Analysis should identify visual facts, motion intent, and continuity rules separately so prompt writing does not have to infer them ad hoc.
- Story analysis should not mention ComfyUI node details, workflow JSON details, or render patch points.

## Planned Commands

- `python -m orchestrator lmstudio-check`
  - verifies LM Studio connectivity, configured model name, and local API reachability
- `python -m orchestrator analyze-scene <project_slug> --scene <scene_id>`
  - reads source material and writes the analysis files for one scene
- `python -m orchestrator analyze-project <project_slug>`
  - optional later convenience command that writes project summary plus all scene-level analysis

## Handoff

- input:
  - local source text under `01_source/`
  - project and scene IDs
  - LM Studio model configuration
- output:
  - analysis files under `02_story_analysis/`
  - no render jobs submitted
  - no prompt packages written yet

## Acceptance

- Prompt writing can trace shot prompts and interval prompts back to stable analysis files.
- One scene can be analyzed with LM Studio running and ComfyUI closed.
- Analysis output names and locations are stable enough that later prompt-writing commands can overwrite or refresh them predictably.
